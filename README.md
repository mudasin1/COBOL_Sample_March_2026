# COBOL Sample for Business Rule Extraction

This sample was generated by Swimm for analysis with rich domain logic. The repository is laid out like a small mainframe-style application: **copybooks**, **COBOL sources**, **integration drivers**, **JCL skeletons**, and **local build/tests** via GnuCOBOL.

## Layout

| Path | Purpose |
|------|---------|
| [cpy/POLDATA.cpy](cpy/POLDATA.cpy) | Shared policy / premium / servicing / claims record (`COPY POLDATA`) |
| [cobol/NB-UW-001.cob](cobol/NB-UW-001.cob) | New business — eligibility, rating, premium, referrals |
| [cobol/SVC-BILL-001.cob](cobol/SVC-BILL-001.cob) | Servicing — grace/lapse, amendments, repricing |
| [cobol/CLM-ADJ-001.cob](cobol/CLM-ADJ-001.cob) | Claims — intake, investigation, adjudication, settlement |
| [cobol/drivers/](cobol/drivers/) | Batch-style drivers that `CALL` the programs and print one result line |
| [jcl/](jcl/) | z/OS JCL templates ([jcl/README.md](jcl/README.md)) |
| [tests/expected/](tests/expected/) | Golden stdout files for integration tests |

## Local build and test (GnuCOBOL)

**Prerequisite:** [GnuCOBOL](https://gnucobol.sourceforge.io/) (`cobc`) on your `PATH` (e.g. macOS: `brew install gnucobol`, Ubuntu: `sudo apt install gnucobol`).

```bash
make all      # compile drivers + business logic into build/bin/
make test     # run drivers and diff stdout vs tests/expected/
make clean    # remove build/
```

Drivers use **fixed `PM-PROCESS-DATE` values** so golden files stay stable (the business programs also call `ACCEPT FROM DATE` only when the process date is zero).

### z/OS vs GnuCOBOL

Sources target IBM-style batch COBOL. For GnuCOBOL, the tree includes small adjustments (line-length / continuation-friendly statements, `EVALUATE`/`MOVE` forms the open compiler accepts, and **face-amount literals** that fit `PIC 9(11)V99` in the copybook). On **Enterprise COBOL** under z/OS, keep your shop’s dialect rules authoritative; use [jcl/COMPILE.jcl](jcl/COMPILE.jcl) as a starting point.

## Domains covered

1. Term Life Policy — New Business and Policy Issuance  
2. Policy Servicing — Policy Changes and Amendments  
3. Term Policy Claims — End-to-End Claims Processing  

## Business rule coverage (summary)

### New Business

- Issue age min and max by plan  
- Sum assured min and max by plan  
- Maturity age checks  
- T65 hazardous occupation restriction  
- Preferred, standard, table, and decline UW logic  
- Gender, smoker, occupation, and UW class rating factors  
- Modal premium calculation with mode factors  
- Rider eligibility and pricing  
- Reinsurance and manual UW referral triggers  

### Servicing

- Payment status evaluation: active, grace, lapse  
- Outstanding premium tracking  
- Plan change validation and repricing  
- Sum assured increase thresholds that trigger underwriting  
- Billing mode changes with modal repricing  
- Rider add/remove flows  
- Reinstatement window and fee logic  
- Premium delta calculation  

### Claims

- Required claim intake fields and documents  
- Contestability investigation trigger  
- Suspicious cause of death investigation triggers  
- Suicide exclusion window  
- Expiry check  
- Accidental death rider settlement  
- Grace premium deduction  
- Policy loan deduction  
- Claim decision and settlement update flow
